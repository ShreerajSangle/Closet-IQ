ClosetIQ — Complete Build Prompt
Context & Vision
Build a full-stack wardrobe management and outfit combination web application called ClosetIQ. This app is built for a specific user — Shreeraj — who has a warm medium skin tone (#CC9674), near-black eyes (#1F1919), and deep black hair (#0A0B0B). These three values are permanently stored in the user profile and actively influence every outfit combination score the app generates. The core purpose of this app is not to suggest outfits in a generic way — it is to act as a personal combination engine that takes every item in the user's wardrobe and mathematically generates every valid outfit pairing based on color harmony, style compatibility, layering rules, pattern logic, and skin tone awareness. The word "suggest" should never appear anywhere in the UI. Use words like "generated", "matched", "combinations", or "pairs with" instead. This is a tool that respects the user's intelligence — it shows them what works and why, not what it thinks they should wear.

Tech Stack
The frontend is built with React and TypeScript, styled using Tailwind CSS, with Zustand for state management and React Router v6 for navigation. The backend is Supabase — using PostgreSQL for the database, Supabase Auth for authentication, and Supabase Storage for clothing item images. The AI explanation layer uses the Groq API (llama3-70b model) but is called only on demand — never automatically — to keep API usage minimal. The font is Satoshi from Fontshare CDN. Icons are from Lucide React. The design system uses warm neutral surfaces (#F7F4EF in light mode, #171614 in dark mode) with a warm teal accent (#01696F). Both light mode and dark mode must be fully supported with a toggle in the header. The UI must be clean, minimal, and mobile-first — designed for 375px screens first and scaling up to 1440px.

Database Schema
Create four tables in Supabase. The first is user_profile which stores id (uuid, references auth.users), display_name (text), skin_tone_hex (text, default #CC9674), eye_color_hex (text, default #1F1919), hair_color_hex (text, default #0A0B0B), skin_tone_type (text, default warm-medium), and style_preferences (text array). The second is wardrobe_items which stores id (uuid), user_id (uuid, references auth.users), name (text), category (text — one of: shirt, tshirt, pants, jeans, shorts, jacket, shoes, accessory), sub_category (text — e.g. camp-shirt, oxford, chinos, leather-jacket, denim-jacket, sneakers), primary_color (text, hex), secondary_color (text, hex, nullable — for patterns), color_family (text — one of: neutral, warm-earth, cool-blue, warm-pink, dark, grey, cream), style_tags (text array — casual, smart-casual, formal, streetwear, statement), occasion_tags (text array — office, weekend, date-night, going-out, gym), pattern (text — solid, stripe, plaid, graphic, texture), season (text array), brand (text), image_url (text, Supabase Storage URL), times_worn (integer, default 0), and last_worn (timestamp). The third is outfits which stores id (uuid), user_id, top_id, bottom_id, shoes_id, jacket_id (nullable), compatibility_score (integer 0–100), occasion_tags (text array), is_saved (boolean), is_favorite (boolean), worn_count (integer), ai_explanation (text, cached Groq response), and created_at. The fourth is outfit_history which stores id, user_id, outfit_id, worn_date (date), occasion (text), and notes (text). Enable Row Level Security on all tables so users only ever see and modify their own data.

Outfit Generation Algorithm
The combination engine lives in src/utils/outfitGenerator.ts. It first separates all wardrobe items into four buckets: tops (shirts and t-shirts), bottoms (pants, jeans, shorts), shoes, and jackets. It then runs a triple nested loop — for every top, for every bottom, for every shoe pair — and calculates a compatibility score. If the score is 60 or above, it is included as a valid outfit. It then runs a fourth loop inside each trio to test every jacket as an optional layer, and if the jacket passes the layering rules check AND the total score is 60 or above, it adds that as a separate layered outfit. The output is sorted descending by score.

The compatibility score is calculated by combining five components. First, color harmony (40% of score): using HSL color space math, calculate the hue distance between the top's primary color and the bottom's primary color. Analogous colors (0–30° apart) score 85. Complementary colors (150–210° apart) score 90. Neutral colors (saturation below 15%) score 95 with everything. Clashing combinations (31–149° apart between two saturated colors) score 50. Second, skin tone affinity (20% of score): items in the warm-earth, cream, and dark color families get a +8 bonus for Shreeraj's warm medium #CC9674 skin tone. Items that are exclusively cool-grey (grey family, low saturation, low warmth) get a -5 penalty when worn head-to-toe with another grey item. Third, style compatibility (20% of score): both items must share at least one style tag to score full points. If a statement/graphic item is paired with a smart-casual item, apply a -20 penalty. Fourth, pattern logic (10% of score): solid + anything = no penalty. Stripe + plaid = -25 penalty. Graphic + graphic = score forced to 0 (blocked entirely). Graphic + plaid = -20 penalty. Stripe + graphic = -15 penalty. Two textures = -10 penalty. Fifth, shoe harmony (10% of score): neutral shoes (cream, white, grey) pair with everything at full points. Dark shoes pair best with dark or earth-tone bottoms. Bright or statement shoes require a neutral top and bottom.

The layering rules in canLayer(top, jacket) work as follows. A leather jacket can layer over any shirt, t-shirt, or camp shirt — always allowed. A denim jacket can layer over any oxford, button-down, t-shirt, or camp shirt — always allowed. A denim jacket over a linen camp shirt applies a -10 penalty (too casual-on-casual). A graphic shirt under any structured jacket applies a -15 penalty. Two jackets together are always blocked. A patterned shirt (stripe or plaid) under a patterned jacket applies a -20 penalty.

Skin Tone, Eye & Hair Color Integration
The user's physical color profile — skin #CC9674, eyes #1F1919, hair #0A0B0B — must be used in three places throughout the app. First, in the outfit score calculation: warm earth tones, creams, dark neutrals, and navy tones are boosted because they complement warm medium tan skin. Cool pastels and washed-out greys are mildly penalized when used as the dominant item. Second, in the Gap Advisor page: when recommending items to buy, the app calculates which colors would score highest given the user's skin tone and prioritizes those in the recommendation list. Specifically recommend: black slim jeans, dark navy chinos, beige/tan chinos, olive trousers, burgundy trousers, white t-shirt, black t-shirt, and white linen shirt — explaining for each one why it works with warm medium skin and the existing wardrobe. Third, in the Profile page: render three color swatch circles side by side labeled Skin, Eyes, and Hair using the actual hex values as background colors. Make these editable so the algorithm recalculates if the user updates their values.

App Pages
The app has six pages. The Dashboard (/) shows a stats row with total wardrobe items, total generated outfits, saved outfits, and last worn date. Below that is a large "Generate My Outfits" button. Below that is a horizontal scroll of recently worn outfits and a quick-view grid of all wardrobe items as small color-dot cards. The Closet (/closet) shows all wardrobe items in a 3-column masonry grid. Each card shows the item image, name, brand, color swatch circle, and style tags as chips. A floating "Add Item" button opens a right slide-over panel with a form for: photo upload to Supabase Storage, name, brand, category, sub-category, primary color hex input with a live color preview circle, secondary color hex input, style tags multi-select, occasion tags multi-select, pattern type, and season. A filter bar at the top allows filtering by category and sorting by recently added, most worn, or color family. The Outfit Generator (/outfits) runs the combination engine on load and displays all valid outfits as cards in a grid. Each card shows a row of four color swatch circles (top, bottom, shoes, jacket or dash), the item names for each slot, the compatibility score as five filled/empty dots plus the number, and three action buttons: Save, Mark as Worn, and "Why this works?" The page has occasion filter tabs at the top: All, Casual, Smart Casual, Office, Date Night, Going Out, Weekend. The Outfit Detail (/outfits/:id) page shows a large layout of the full outfit with a circular color harmony wheel built from the item hex codes, the AI explanation panel (loaded on demand via Groq), a "Mark as Worn" button that logs to outfit_history and increments worn_count, and a section of similar scored outfits. The Gap Advisor (/gaps) analyzes the current wardrobe and shows: items you have vs. category gaps, how many new outfit combinations each suggested item would unlock, and a prioritized shopping list sorted by outfit impact — always filtered through the user's skin tone compatibility. The Profile (/profile) shows the three physical color swatches (skin, eyes, hair) as editable hex inputs with live preview circles, style preference chips, and wardrobe stats.

Groq AI Integration
The Groq API is called exclusively from src/utils/groqExplainer.ts and only when the user explicitly clicks "Why this works?" on an outfit card. The response is cached in the ai_explanation column of the outfits table so the API is never called twice for the same outfit. The prompt sent to Groq must be: "You are a fashion stylist. In exactly 2 sentences, explain why this outfit works for someone with warm medium tan skin (hex: #CC9674), near-black eyes (#1F1919), and deep black hair (#0A0B0B). Top: [name, color hex, pattern]. Bottom: [name, color hex]. Shoes: [name, color hex]. Jacket: [name, color hex or 'none']. Focus on color harmony, contrast, layering logic, and how the colors interact with the person's complexion. Be specific, confident, and direct. No generic fashion advice." Stream the response into a modal or tooltip that appears below the outfit card.

UI & Design Rules
The design must feel like a premium personal styling tool — clean, warm, and confident. Use Satoshi font loaded from https://api.fontshare.com/v2/css?f[]=satoshi@400,500,700&display=swap. Surface colors are warm off-white in light mode and near-black in dark mode. The single accent color is warm teal #01696F used only on primary CTAs and active states. No gradient buttons — solid accent color only. No colored left borders on cards. Cards use subtle shadow and 12px border radius. Outfit scores are shown as five dot indicators (filled teal dots for score, empty grey dots for remainder), never as a progress bar. Color swatches throughout the UI are 16px circles with the actual item hex color as background and a 1px semi-transparent border. Empty states must have a warm illustrative message and a clear CTA — never just "No items found." All slide-over panels animate from the right at 200ms ease-out. Hover states on cards elevate the shadow slightly. Every interactive element has a visible focus ring for accessibility. The mobile layout at 375px collapses the grid to a single column, hides the sidebar, and uses a bottom tab bar for primary navigation.
